마크업 규칙

문법

  • doctype을 포함하여 태그를 대문자로 사용 금지
  • 들여쓰기는 2점식을 사용(2칸 공백)
  • 중첩된 요소는 들여쓰기 한번 사용
  • 속성에는 항상 큰 따옴표를 사용하고 작은 따옴표는 사용 금지
  • 셀프 클로즈 요소(img, input 등)에는 닫는 슬래시 사용 금지

속성 순서

  • HTML 속성은 코드를 더 쉽게 읽을 수 있도록 특정 순서로 제공되어야 합니다.
  • 1. class
  • 2. id, name
  • 3. data-*
  • 4. src, for, type, href, value
  • 5. title, alt
  • 6. role, aria-*

Language 속성

  • html태그에 lang속성을 지정하는것이 좋음
  • lang속성을 지정하지 않으면 음성도구, 번역도구에서 불이익을 받음

IE 호환 모드

  • internet Explorer는 문서 호환성 <meta> 태그 사용을 지원하여 페이지를 렌더링 할 IE 버전을 지정합니다
    상황에 따라 달리 필요하지 않는 한 IE에 최신 지원 모드를 에지 모드와 함께 사용하도록 지시하는 것이 가장 유용합니다

문자 인코딩

  • 명시적 문자 인코딩을 선언하여 빠르고 쉽게 콘텐츠의 적절한 렌더링을 보장합니다
  • 인코딩이 문서의 인코딩과 일치한다면 (일반적으로 UTF-8) HTML에서 문자 엔티티를 사용하지 않을 수 있습니다
