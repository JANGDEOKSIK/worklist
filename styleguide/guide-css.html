CSS 규칙

문법

  • 들여쓰기는 2점식을 사용(2칸 공백)
  • 중첩된 요소는 들여쓰기 한번 사용
  • 가독성을 위해 선언 블록의 여는 중괄호 앞에 하나의 공백을 포함합니다
  • 각 선언마다 : 뒤에 하나의 공백을 포함합니다
  • 세미콜론으로 모든 선언을 끝냅니다. 마지막 속성 뒤에도 세미콜론을 붙여줍니다.
  • 쉼표로 구분 된 속성 값은 각 쉼표 뒤에 공백을 포함해야합니다 (예 : box-shadow)
  • 속성 값이나 색상 매개 변수 앞에 0을 붙이지 마세요 (예 : 0.5 대신 .5, -0.5px 대신 -.5px)
  • 모든 16 진수 값을 소문자로 지정합니다 (예 : #fff)
  • 가능한 경우 속기 16 진수 값을 사용합니다 (예 : #ffffff 대신 #fff)
  • 메일폼에서는 속기 16 진수가 아닌 #ffffff으로 사용합니다.
  • 값에 대한 단위를 지정하지 마십시오 (예 : margin: 0)

속성 순서

  • 관련 속성 선언은 순서에 따라 함께 그룹화되어야합니다.
  • 1. Positioning
  • 2. Box model
  • 3. Typographic
  • 4. Visual
  • 5. 기타

@import 사용금지

  • <link>에 비해 @import는 더 느리고 추가 페이지 요청을 추가하며 다른 예상치 못한 문제를 일으킬 수 있습니다. 이를 피하고 대신 대체 접근 방식을 선택하십시오.

미디어쿼리 배치

  • 가능한 한 관련 규칙 세트에 가깝게 미디어 쿼리를 배치합니다

문자 인코딩

  • 명시적 문자 인코딩을 선언하여 빠르고 쉽게 콘텐츠의 적절한 렌더링을 보장합니다
  • 인코딩이 문서의 인코딩과 일치한다면 (일반적으로 UTF-8) HTML에서 문자 엔티티를 사용하지 않을 수 있습니다
